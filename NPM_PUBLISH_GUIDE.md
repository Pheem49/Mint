# 🚀 NPM Publishing Guide for Mint

This guide summarizes how to update and publish the `@pheem49/mint` package to the NPM Registry.

## 1. Before You Publish
Every time you want to release a new version, you **must** update the version number in `package.json`. NPM will not allow you to publish over an existing version.

- Open `package.json`
- Update `"version": "1.2.x"` (increment the last digit for small fixes).

## 2. Standard Publish Flow
Run the following commands in your terminal:

```bash
# 1. Login to your NPM account (if not already logged in)
npm login

# 2. Publish the package
npm publish --access public
```

## 3. Handling 2FA (Security Key / OTP)
Since your account has 2FA enabled, you will be prompted to authenticate:

- **If using an Authenticator App**:
  ```bash
  npm publish --access public --otp=123456
  ```
- **If using a Security Key**:
  NPM will provide a link in the terminal. Open it, authenticate on the website, and the terminal will automatically finish the upload.

## 4. Maintenance Commands
- **Check current version**: `npm view @pheem49/mint version`
- **Install locally for testing**: `sudo npm link`
- **Verify global installation**: `mint --version`

## 5. Common Issues
- **403 Forbidden**: Usually means you forgot the `--otp` code or your session expired. Run `npm login` again.
- **Version Conflict**: You tried to publish a version that already exists on NPM. Change the version in `package.json`.
